# Security Policy

This repository is intentionally public-safe. It must not contain production
tenant data, credentials, API keys, private screenshots, account IDs, personal
data, or real secret values.

If you find sensitive material in this repository, open a private issue or
contact the repository owner directly instead of posting the value publicly.

## Public-Safe Rules

- Use synthetic or sanitized event data only.
- Keep Datadog tenant names, user emails, account IDs, and API paths out of Git.
- Do not commit raw exports until they have been reviewed and scrubbed.
- Do not commit API keys, application keys, session cookies, or auth headers.
- Keep screenshots generated or redacted before publication.
